Agent 助手

一个面向电磁、信号处理与毫米波资料问答场景的本地知识检索应用。仓库证据显示，它以 React 聊天界面承载交互，由 FastAPI 调用 LangChain Agent，并通过 MCP 工具对 Milvus 做稠密 + 稀疏混合检索后返回答案。

仓库名：Agent_System

它是做什么的

这是一个“聊天式检索助手”，不是通用门户或多页面业务系统。前端欢迎语与示例问题都指向电磁波、信号处理、5G 毫米波等主题，说明它主要围绕专业文献/知识查询使用。

它面向谁

AI

主要面向需要用中文快速查询相关技术资料的研究人员、学生或工程人员，尤其是电磁、通信、信号处理、毫米波方向用户。

它能做什么

  • 提供单页聊天界面，支持输入问题、显示问答记录与清空会话。
  • 前端以 thread_id 维持当前会话线程，并通过 /api/chat 发起请求。
  • 后端暴露聊天接口与健康检查接口，统一返回整理后的完整回复。
  • LangChain Agent 通过 MCP 挂载 milvus_hybrid_search 检索工具。
  • 检索同时使用稠密向量与稀疏向量，并以 RRF 重排结果。
  • 返回结果包含标题、作者、年份与正文内容片段，适合资料问答。
  • 附带离线数据管线：PDF 提取 → Excel → 向量文件 → Milvus 入库。

它如何工作

在线链路由 5 个部分组成：前端 UI、FastAPI 网关、LangChain Agent、MCP 检索服务、Milvus/向量服务。

用户
→ React + Vite 前端（3000）
→ POST /api/chat 代理到 FastAPI（8000）
→ Agent 调用本地 MCP Server（stdio）
→ MCP 先向 54331/54332 取稠密/稀疏向量，再访问 Milvus（19530）执行 hybrid_search + RRF
→ 返回 Top 结果给 Agent
→ 后端汇总消息流后返回前端显示

离线链路：Data/*.pdf 经 RAG/Get_data_from_pdf.py 导出到 pdf_extract_result.xlsx，再由 RAG/Loading_in_Milvus.py 分块、生成向量并写入 Agent_System 集合。

如何运行

  • 准备本地依赖服务：Milvus 需监听 http://localhost:19530；聊天/向量接口需监听 54329、54331、54332。这些服务的启动脚本与部署说明：Not found in repo.
  • 启动后端：在仓库根目录执行 python Back_end/api_server.py。
  • 启动前端：进入 Front_end 后执行 npm install、npm run dev。
  • 打开 http://localhost:3000；前端会把 /api 请求代理到 http://localhost:8000。
  • Python 依赖清单文件（如 requirements.txt / pyproject.toml）：Not found in repo.
